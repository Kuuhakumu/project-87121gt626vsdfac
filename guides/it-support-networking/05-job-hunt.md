## honest entry reality

this is the IT field that genuinely hires from zero. A+ + documented home lab + decent communication is enough for many first help-desk roles.

```text
Help Desk T1 / Service Desk Analyst
  -> Desktop Support / NOC Technician / Junior Sysadmin
  -> Network Admin / Sysadmin / Cloud or Security pivot
  -> Network/Systems Engineer / Cloud Engineer / Security Analyst
```

target the first row for your first job. help desk is the launchpad.

AI is compressing pure T1 work at large enterprises, so position above pure ticket-handling: escalations, scripts, cloud/identity skills. MSPs and smaller orgs are still strong entry grounds.

## resume and profiles

- [ ] lead with certs + home lab
- [ ] include customer-facing experience
- [ ] quantify: "Built a Windows domain lab with DC + 3 clients"
- [ ] mirror posting keywords: Active Directory, Office 365, ticketing, troubleshooting, TCP/IP
- [ ] link GitHub and pin lab write-ups
- [ ] headline example: `Aspiring IT Support Specialist | CompTIA A+ | AD & PowerShell home lab`

soft skills are real filters here. help desk is customer service with tech attached.

## job titles to search

true-entry:

- Help Desk Technician
- Service Desk Analyst
- IT Support Specialist
- Desktop Support Technician
- NOC Technician
- Technical Support Representative

next step:

- Junior Systems Administrator
- Junior Network Technician
- Field Service Technician

dont skip help desk waiting for "Network Engineer." that comes after.

## find and target roles

- [ ] prioritize MSPs; they hire T1 in volume and expose u to many environments
- [ ] list 20-40 target employers
- [ ] include healthcare, schools, universities, government contractors, mid-size orgs
- [ ] check careers pages directly
- [ ] use referrals through r/sysadmin, r/ITCareerQuestions, local IT meetups
- [ ] filter remote for help-desk/NOC; desktop/field/network tech is often on-site
- [ ] tailor each application with relevant home-lab link

## interview prep

see [interview-prep.md](interview-prep.md).

questions u will be asked:

- [ ] user cant connect to internet - troubleshoot from layer 1
- [ ] what happens when typing a URL
- [ ] DHCP vs static IP

- [ ] hub vs switch vs router
- [ ] hosts in `/26`

behavioral prompts:

- [ ] angry user yelling about a problem
- [ ] u dont know the answer
- [ ] explain technical thing to non-technical person
