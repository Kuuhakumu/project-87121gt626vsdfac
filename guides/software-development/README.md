## how to track your progress meow

oki so u might notice all the `- [ ]` checkboxes in this guide... heres the thing: **these boxes are display-only in this public repo** - u cant actually tick them here (youd have to commit to *our* repo which... doesnt make sense for your personal progress ;w;).

**so heres what to do instead:**
- **fork this repo** into your own GitHub account - then u can tick boxes in your fork and commit your progress there, or
- **copy the checklist into your own notes** - if youre using Obsidian or OneNote, just paste the checkbox tree into a page there and tick as u go, or
- **download/print it** - old school but it works meow

the `- [ ]` boxes in *this* repo are just the visual structure showing u the path.
your real progress is the gate at the end of each goal: the project works, the lab is passed, or u can explain the work without hiding behind a tutorial.

## how the hour system works

timelines are in **study hours**, not weeks, so they work at any pace.

| Your pace | 500 hours takes |
|---|---|
| 1 hr/day | ~16 months |
| 2 hrs/day | ~8 months |
| 4 hrs/day | ~4 months |
| 6 hrs/day (full-time) | ~3 months |

each goal shows an approximate hour band. thats a budget, not a deadline. at the default **2 h/day**, this whole path is roughly **8 months**. dw, slow and steady is the point.

## read this first: certs barely matter here

unlike cybersecurity or cloud, **software dev hiring runs on your portfolio and whether u can pass a coding interview - not certs.**

- [ ] **spend money carefully** - domain name, cheap hosting, maybe one paid course.
- [ ] **spend time where it counts** - deployed projects, clean GitHub, DSA practice, explaining trade-offs.
- [ ] **treat certs as optional** - cloud certs can help for cloud-adjacent roles; see [certifications.md](certifications.md).

a deployed project u can explain beats almost any certificate. thats the whole game here meow.

## pick your track later, not rn

| Track | youll like it if... | Core stack (2026) |
|---|---|---|
| **Frontend** | u care about UI, design, what users see | HTML/CSS, JS -> TS, React, Next.js, Tailwind |
| **Backend** | u like data, logic, systems | Python (FastAPI/Django) or Node, PostgreSQL, REST |
| **Full-Stack** | u want to build whole things | Next.js + a backend + Postgres |
| **Mobile** | u want apps in peoples pockets | Kotlin, Swift, React Native, or Flutter |

Goals 0-3 are shared for everyone. u only branch at [Goal 4](04-specialization.md), after u have enough context to choose without guessing.

## what makes this guide different

- **hour-based** - fits any schedule, not rigid weeks.
- **verified June 2026** - stack choices checked against current research notes.
- **current stack** - React + Next.js, FastAPI, PostgreSQL, Vite, Docker, TypeScript-by-default. no CRA, no jQuery-first, no class-component-first path.
- **AI-realistic** - use AI tools with judgment, bc employers screen for understanding now.
- **free-first** - the core path can be done through freeCodeCamp, The Odin Project, Full Stack Open, and CS50.
