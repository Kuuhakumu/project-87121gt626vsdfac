## books

| Book | Author | Why |
|---|---|---|
| *The Pragmatic Programmer* | Hunt & Thomas | timeless craft and mindset |
| *Clean Code* | Robert C. Martin | naming and functions; read critically |
| *Cracking the Coding Interview* | Gayle Laakmann McDowell | DSA interview standard |
| *Eloquent JavaScript* | Marijn Haverbeke | deep, free JS book ([online](https://eloquentjavascript.net/)) |
| *You Don't Know JS Yet* | Kyle Simpson | free, deep JS series on GitHub |
| *Grokking Algorithms* | Aditya Bhargava | illustrated beginner-friendly DSA |
| *Designing Data-Intensive Applications* | Martin Kleppmann | backend depth for year 2+ |

## free deployment and sandbox

- [ ] **Deploy frontend:** [Vercel](https://vercel.com/), [Netlify](https://www.netlify.com/)
- [ ] **Deploy API/full-stack:** [Railway](https://railway.app/), [Render](https://render.com/), [Fly.io](https://fly.io/)
- [ ] **Database:** [Supabase](https://supabase.com/), [Neon](https://neon.tech/)
- [ ] **Cloud IDE:** [GitHub Codespaces](https://github.com/features/codespaces), [StackBlitz](https://stackblitz.com/), [Replit](https://replit.com/)

Heroku killed its free tier in 2022. use the options above instead; older guides still recommend Heroku.

## communities

- [ ] **Reddit:** r/learnprogramming, r/cscareerquestions, r/webdev, r/ExperiencedDevs
- [ ] **Discord:** The Odin Project, freeCodeCamp, Reactiflux, language-specific servers
- [ ] **Q&A:** Stack Overflow, GitHub Discussions
- [ ] **Open source:** search GitHub for `good first issue`

communities help when u ask specific questions. they hurt when u use them to compare timelines all day, so be careful meow.

## daily tools

- [ ] **Editor:** [VS Code](https://code.visualstudio.com/)
- [ ] **AI assist:** GitHub Copilot, Cursor, or Claude - boilerplate only unless u understand every line
- [ ] **Package managers:** npm/pnpm, pip/uv
- [ ] **Build tool:** Vite, not deprecated Create React App
- [ ] **API testing:** Postman, Insomnia, or `curl`
