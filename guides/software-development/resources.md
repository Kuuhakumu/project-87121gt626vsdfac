## 📚 Books

| Book | Author | Why |
|---|---|---|
| *The Pragmatic Programmer* | Hunt & Thomas | Timeless craft and mindset |
| *Clean Code* | Robert C. Martin | Readable code, naming, functions (read critically) |
| *Cracking the Coding Interview* | Gayle Laakmann McDowell | The DSA interview standard |
| *Eloquent JavaScript* | Marijn Haverbeke | Deep, free JS book ([online](https://eloquentjavascript.net/)) |
| *You Don't Know JS Yet* | Kyle Simpson | Free, deep JS series on GitHub |
| *Grokking Algorithms* | Aditya Bhargava | Illustrated, beginner-friendly DSA |
| *Designing Data-Intensive Applications* | Martin Kleppmann | Backend depth (year 2+) |

## ☁️ Free deployment & sandbox

- **Deploy:** [Vercel](https://vercel.com/), [Netlify](https://www.netlify.com/) (frontend); [Railway](https://railway.app/), [Render](https://render.com/), [Fly.io](https://fly.io/) (full-stack/API)
- **Database:** [Supabase](https://supabase.com/), [Neon](https://neon.tech/) (free Postgres tiers)
- **Cloud IDE:** [GitHub Codespaces](https://github.com/features/codespaces), [StackBlitz](https://stackblitz.com/), [Replit](https://replit.com/)

> ⚠️ **Heroku killed its free tier** (2022). Use the options above instead — older guides still recommend Heroku.

## 👥 Communities

- **Reddit:** r/learnprogramming, r/cscareerquestions, r/webdev, r/ExperiencedDevs

## 🔧 Daily tools

- **Editor:** [VS Code](https://code.visualstudio.com/) (75.9% of developers use it)
- **AI assist:** GitHub Copilot, Cursor, or Claude — use for boilerplate, but understand every line (see [Phase 0](00-prep.md))
- **Package managers:** npm/pnpm (JS), pip/uv (Python)
- **Build tool:** Vite (not the deprecated Create React App)
- **API testing:** Postman, Insomnia, or `curl`
