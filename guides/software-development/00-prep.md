# ⚙️ Phase 0: Pre-Flight Setup

> **Day 1 — non-negotiable.** Just sign up and install — no learning to finish, do it before Phase 1.
> Goal: working dev environment, the right mindset, a chosen track — before you write real code.

## Mindset & expectations

- **Portfolio beats certs in this field.** Unlike cybersecurity or cloud, software employers hire on what you can *build*. A GitHub with real, deployed projects you can explain outweighs almost any certificate. (See [certifications.md](certifications.md) for the short list of exceptions.)
- **You don't need a CS degree** — but you do need to cover what a degree covers: fundamentals, data structures, and the ability to explain your decisions. Self-taught and bootcamp grads get hired every year; the ones who go deep on backend/databases/systems tend to do best.
- **Consistency beats intensity.** Two focused hours daily beats a 12-hour weekend once a month.
- **Build, don't just watch.** Tutorial-watching feels like progress but isn't. Write code, break it, fix it. The "tutorial trap" is the #1 reason beginners stall.
- **AI tools are part of the job now — but judgment is the skill.** Use Copilot/Cursor/Claude for boilerplate and first drafts, but you need to understand every line. Interviewers can tell when you can't explain your own code, and it disqualifies candidates.

## Pace & timeline

Measured in **study hours**, not weeks, so it works at any pace. Total to job-ready: **~400–600 hours**.

| Pace | Hours/week | ~Time to job-ready |
|---|---|---|
| Casual | 5–8 | 14–20 months |
| Part-time (recommended) | 10–15 | 9–14 months |
| Full-time | 20–30 | 5–8 months |

## Set up your environment (~6 hrs)

A real dev environment from day one — no "learn in a browser sandbox forever."

1. **Editor:** [VS Code](https://code.visualstudio.com/) (free, 75% of developers use it). Install extensions for your language as you go.
2. **Terminal comfort:** learn to open and use your terminal (Bash/zsh on macOS/Linux, PowerShell or WSL on Windows). [The Missing Semester (MIT)](https://missing.csail.mit.edu/) is the free gold standard.
3. **Git + GitHub:** install Git, create a [GitHub](https://github.com/) account. This is your portfolio's home — every project lands here.
4. **Runtime for your track:** Node.js (JS/TS) via [nvm](https://github.com/nvm-sh/nvm), or Python via [uv](https://github.com/astral-sh/uv)/python.org.

> **Zero-setup alternative for week 1:** if local setup is blocking you, start in a cloud IDE — [Replit](https://replit.com/), [GitHub Codespaces](https://github.com/features/codespaces) (60 free core-hrs/mo), or [Gitpod](https://gitpod.io/). But move to local by Phase 1 — environment skills are tested.

## Pick your track and first language (~8 hrs)

You don't have to commit to a track forever, but pick a direction so you're not learning everything at once.

| Track | First language | Then | Good if you like… |
|---|---|---|---|
| **Frontend** | JavaScript → TypeScript | React, CSS | Visual feedback, UI/UX, design |
| **Full-Stack** | JavaScript → TypeScript | Node + React, then DB | Doing a bit of everything |
| **Backend** | Python | FastAPI/Django, SQL | Logic, data, systems |
| **Data / ML / AI** | Python | NumPy, pandas, then ML | Math, data, experiments |
| **Mobile** | Kotlin (Android) / Swift (iOS) | Jetpack Compose / SwiftUI | Phone apps, polished UX |

> **Not sure? Default to JavaScript/TypeScript (full-stack)** — most entry-level postings, especially at startups, are full-stack JS roles. **Or Python** if you lean toward data/AI or find JS frustrating. Both are excellent first languages; the choice is about your *goal*, not which is "better."

## Plug in & get oriented (~6 hrs)

- **Pick a structured path now** (you'll start it in Phase 1): [The Odin Project](https://www.theodinproject.com/) (full-stack JS, free), [freeCodeCamp](https://www.freecodecamp.org/) (free), or [CS50x](https://cs50.harvard.edu/x/) (Harvard, free, best for fundamentals).
- **Follow the map:** bookmark [roadmap.sh](https://roadmap.sh/) — interactive visual roadmaps for every track. Use it to see what's ahead and check off topics.
- **Communities:** r/learnprogramming, r/cscareerquestions (read the wiki first), The Odin Project Discord, freeCodeCamp forum.
- **Notes:** set up a system ([Obsidian](https://obsidian.md/), Notion, or plain markdown in a Git repo). Document what you learn — it compounds.
