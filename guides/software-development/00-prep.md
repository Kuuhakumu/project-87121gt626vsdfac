## understand the field honestly (~1h)

- **Portfolio beats certs here.** unlike cybersecurity or cloud, software employers hire on what u can *build*. a GitHub with real, deployed projects u can explain outweighs almost any certificate.
- **u dont need a CS degree** - but u do need the fundamentals, data structures, and the ability to explain decisions.
- **consistency beats intensity.** two focused hours daily beats a 12-hour weekend once a month.
- **build, dont just watch.** tutorial-watching feels like progress but isnt. write code, break it, fix it.
- **AI tools are part of the job now, but judgment is the skill.** use Copilot/Cursor/Claude for boilerplate and first drafts, but understand every line. interviewers can tell when u cant explain your own code.

dw, this isnt saying "be perfect before starting." its saying dont mistake watching for doing meow.

## pace and timeline

measured in **study hours**, not weeks, so it works at any pace. total to job-ready: **~400-600 hours**.

| Pace | Hours/week | ~Time to job-ready |
|---|---|---|
| Casual | 5-8 | 14-20 months |
| Part-time (recommended) | 10-15 | 9-14 months |
| Full-time | 20-30 | 5-8 months |

## set up your environment (~6h)

a real dev environment from day one. dont live in browser sandboxes forever.

- [ ] **Editor:** install [VS Code](https://code.visualstudio.com/) (free). add language extensions as u go.
- [ ] **Terminal:** learn to open and use Bash/zsh on macOS/Linux, PowerShell or WSL on Windows. [The Missing Semester (MIT)](https://missing.csail.mit.edu/) is the free gold standard.
- [ ] **Git + GitHub:** install Git and create a [GitHub](https://github.com/) account. this is your portfolio home.
- [ ] **Runtime for your track:** Node.js (JS/TS) via [nvm](https://github.com/nvm-sh/nvm), or Python via [uv](https://github.com/astral-sh/uv)/python.org.

> **zero-setup fallback for week 1:** if local setup blocks u, start in [Replit](https://replit.com/), [GitHub Codespaces](https://github.com/features/codespaces) (60 free core-hrs/mo), or [Gitpod](https://gitpod.io/). but move local by Goal 1, bc environment skills get tested.

## pick your track and first language (~8h)

u dont have to commit forever. just pick a direction so u arent learning everything at once.

| Track | First language | Then | Good if u like... |
|---|---|---|---|
| **Frontend** | JavaScript -> TypeScript | React, CSS | visual feedback, UI/UX, design |
| **Full-Stack** | JavaScript -> TypeScript | Node + React, then DB | doing a bit of everything |
| **Backend** | Python | FastAPI/Django, SQL | logic, data, systems |
| **Data / ML / AI** | Python | NumPy, pandas, then ML | math, data, experiments |
| **Mobile** | Kotlin / Swift | Jetpack Compose / SwiftUI | phone apps, polished UX |

not sure? default to **JavaScript/TypeScript full-stack** for broad entry-level coverage, or **Python** if u lean data/AI or find JS frustrating. both are fine first languages; the choice is about your goal, not which one is "better."

## plug in and get oriented (~6h)

- [ ] **Pick one structured path to start in Goal 1:** [The Odin Project](https://www.theodinproject.com/) (full-stack JS, free), [freeCodeCamp](https://www.freecodecamp.org/) (free), or [CS50x](https://cs50.harvard.edu/x/) (Harvard, free, best for fundamentals).
- [ ] **Bookmark [roadmap.sh](https://roadmap.sh/).** use it to see whats ahead and check off topics.
- [ ] **Join communities:** r/learnprogramming, r/cscareerquestions (read the wiki first), The Odin Project Discord, freeCodeCamp forum.
- [ ] **Set up notes:** [Obsidian](https://obsidian.md/), Notion, or plain markdown in a Git repo. document what u learn; it compounds.
