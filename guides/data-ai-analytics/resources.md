## Books worth owning

| Book | For |
|---|---|
| *Storytelling with Data* — Cole Nussbaumer Knaflic | Every analyst — communicating with charts |
| *SQL for Data Analysis* — Cathy Tanimura | Analytical SQL patterns |
| *Python for Data Analysis* — Wes McKinney (pandas creator) | pandas, the canonical reference |
| *Fundamentals of Data Engineering* — Reis & Housley | The DE mental model, tool-agnostic |
| *Designing Data-Intensive Applications* — Kleppmann | Deeper systems (DE, later) |
| *The Analytics Engineering with dbt* / dbt docs | AE track |
| *Hands-On Machine Learning* — Géron | DS/ML track, sklearn + Keras |

> You don't need all of these. Pick the one that fits your track plus *Storytelling with Data*.

## Stay current (the stack moves fast)

- **dbt Developer Blog** and **dbt release notes** — analytics engineering changes
- **Hugging Face blog** — GenAI/LLM tooling
- **DuckDB blog**, **Polars releases** — the fast-mover dataframe/analytics tools
- **Seattle Data Guy / Data Engineering Weekly** newsletters

> **On AI/LLMs:** as a junior, *use* them to write/debug SQL and draft analysis, but make sure you can read and verify the output. Judgment beats generation. Don't try to master LangChain internals — the API changes weekly. Understand RAG and vector DBs at a concept level; leave production LLM pipelines to senior/ML roles.
