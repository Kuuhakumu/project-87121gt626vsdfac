## honest progression

```text
Data / BI Analyst or Analytics Engineer
  -> Senior Analyst / Senior AE
  -> Data Engineer / Data Scientist
  -> Data Architect / ML Engineer / Applied Scientist / Lead / Manager
```

most people enter as Analyst or Analytics Engineer and branch after 1-2 years once they know which problems they enjoy: pipelines, modeling, or business decision work.

## what senior adds by track

### Analyst -> Senior Analyst / Analytics Lead

- owning a metric/domain end to end
- defining metrics and single source of truth
- mentoring juniors and influencing decisions
- experiment design, not just reading A/B tests

### Analytics Engineer -> Senior AE

- data modeling at scale
- semantic layers, large dbt projects, performance tuning
- data quality, testing strategy, CI/CD for warehouse
- warehouse cost optimization
- team conventions and model reviews

### Data Engineer -> Senior DE -> Data Architect

- streaming systems beyond batch
- distributed systems depth
- platform design: ingestion -> storage -> serving
- reliability: freshness SLAs, lineage, observability, data contracts

### Data Scientist -> Senior DS -> ML Engineer

- productionizing models
- deployment, monitoring, drift detection
- ML system design and MLOps
- causal inference and experimentation at scale
- ML Engineer = SWE-heavy serving infra, latency, scale

## GenAI / LLM frontier

expect these to become normal:

- [ ] RAG pipelines over company data
- [ ] vector databases such as pgvector
- [ ] LLMs in pipelines for classification, extraction, summarization
- [ ] evaluation of LLM outputs
- [ ] AI/ML platform roles for serving, prompts, cost control

the junior skill that ages well is **judgment over generation**. anyone can prompt; value is knowing whether output is right.

## skills that compound

| Skill | Why it pays off |
|---|---|
| **Communication / storytelling** | separates senior analysts from query-writers |
| **SQL depth** | every track, every level |
| **One cloud platform deep** | DE/architecture roles demand it |
| **Data modeling** | transfers across AE, DE, architecture |
| **Statistics intuition** | DS/experimentation moat |
| **Git + software habits** | turns analyst into analytics engineer |

## keep growing

- [ ] go deep on one warehouse/platform
- [ ] read other peoples dbt projects, pipeline repos, Kaggle notebooks
- [ ] write project explanations publicly
- [ ] contribute to open source: dbt packages, Polars/pandas issues, Airflow providers
- [ ] pick next cert based on actual track, not badge collecting

the tooling churns. SQL, statistics, data modeling, and clear communication dont churn. invest there.
